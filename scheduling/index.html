Algorithm Encyclopedia

FCFS

NP

Logic: Standard FIFO execution. First arrival maintains CPU control until burst completion.

  • Problem: Convoy Effect (Short jobs stuck behind long ones).
  • Metric: High Average Waiting Time (AWT).
  • Use Case: Batch systems with low interactivity.

SJF / SRTF

NP / P

Logic: Prioritizes processes with the smallest CPU burst requirement.

  • Efficiency: Mathematically optimal for minimum AWT.
  • Risk: Potential starvation for long-duration tasks.
  • SRTF: Preemptive version that checks arrivals at every unit.
Technical Specs →

Priority

NP / P

Logic: CPU is allocated based on assigned rank (Integer-based priority).

  • Constraint: High priority interrupts low priority (Preemptive).
  • Aging: Required to prevent low-priority jobs from never running.
  • Use Case: Real-time and mission-critical kernels.
Technical Specs →

Round Robin

P

Logic: Cyclic execution using a fixed Time Quantum (δ) or slice.

  • Fairness: No starvation; every job gets a slice of time.
  • Responsiveness: Excellent for interactive time-sharing systems.
  • Overhead: Performance depends on Context Switch vs Quantum size.
Technical Specs →

MLQ

Fixed Tiers

Logic: Partitions ready queue into separate tiers (System, User, Batch).

  • Hierarchy: Strict dominance (Q0 > Q1 > Q2).
  • Stability: Tasks never change their assigned queue level.
  • Optimization: Different algorithms can run on different tiers.
Technical Specs →

MLFQ

Adaptive P

Logic: Dynamic migration of tasks based on execution history/behavior.

  • Learning: Demotes CPU-hogs and favors I/O-bound tasks.
  • Goal: Minimizes Response Time without prior burst knowledge.
  • Industry: Standard in modern Unix/Windows NT kernels.
