PRODUCER-CONSUMER
Producer-Consumer Problem
A classic synchronization problem: producers create items and place them in a bounded buffer; consumers remove and use them. Semaphores prevent race conditions and buffer over/underflow.
producer():
  loop:
    wait(empty)
    wait(mutex)
    buffer[in] = item
    in = (in+1) % N
    signal(mutex)
    signal(full)
consumer():
  loop:
    wait(full)
    wait(mutex)
    item = buffer[out]
    out = (out+1) % N
    signal(mutex)
    signal(empty)
SEMAPHORES
Semaphore Operations
wait(S): if S > 0, decrement S; else block until S > 0. signal(S): increment S and wake one blocked process. Mutex is a binary semaphore ensuring mutual exclusion in the critical section.
mutex protects the buffer, empty tracks free slots, and full tracks filled slots.
IPC MECHANISMS
IPC Mechanisms
Shared Memory: fastest, processes map the same memory region. Message Queues: kernel-buffered FIFO messages. Pipes: sequential byte streams with simple unidirectional flow.
Throughput: Shared > Message Queue > Pipe
